Divine-right

An open source, procedurally generated RPG.

The idea

The idea behind Divine Right is that of an RPG in a completely dynamic world. There is no fixed plot, no fixed quests, no scripted sequences and no fixed world or map. The world is acted upon by a number of NPCs who all make their own decisions, and help shape the world. And the player character is in the midst of it all...

The main concept here is that, contrary to other RPGs in which the player character is the most important being on the planet, and the world revolves around him (and the plot waits for him) - NPCs are just as powerful as the player. The player has an underlying mission, but how he does it, and how NPCs will react to it is up to them.

For instance, if a bandit leader takes up home in a town's mine, he has done so for a reason. The town mayor might offer the job to the Player character, or any number of NPCs. If the player character takes the job, there is no guarantee that the bandit will not be killed by someone else in the meanwhile, or die of sickness, or leave of his own volition. And if nobody takes up the mantle, the town's economy could suffer for it.

Done so far (Development branch)

  • World Generates procedurally [Version 1]
  • Local Map Generation based on a number of maplets [Version 1]
  • Local Map Generation based on maplets read from xml file [Version 1]
  • The underlying code for moving around and interacting with objects and object classes
  • Dungeon Generation
  • Enemy Generation including basic combat system and attributes
  • Settlement Generation based on maplets and settlement buildings [Version 2]
  • Settlement location based on civilisation spread
  • Bandit Camp Generation
  • Ability to wander around the world map and visit places
  • Save/Load game functionality
  • Ability to trade with Merchants
  • Inventory
  • Day/Night display cycle
  • Basic Skills
